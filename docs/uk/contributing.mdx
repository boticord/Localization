# Вступ

Будь-хто може зробити свій внесок у розвиток проєкту. Для цього необхідно дотримуватися таких правил:

## Основне

1.  Усі зміни мають бути чітко описані в коміті та pull request'е.
2.  Поводьтеся пристойно. Не використовуйте нецензурну лексику і не ображайте інших учасників проєкту.
3.  Бажано користуватися англійською мовою, оскільки є міжнародною мовою

## Вітки

-   `main` - основна вітка проєкт
-   `dev` - вітка розробки

## Коміти

Чітких правил для комітів немає, але є кілька рекомендацій:

-   Коміти повинні бажано бути англійською мовою
-   Використовувати [conventional commits](https://conventionalcommits.org)

### Приклади

1.  <strike>Changed text</strike> ❌ Коміт не описує, чому змінено текст, де.
2.  **chore(Faq): Added additional answer to the question ** ✅ Ідеально!

## Документація

Документація має бути написана російською мовою. Для написання документації використовується [Docusaurus](https://docusaurus.io/).
Якщо ви хочете змінити документацію іншою мовою, то використовуйте [Crowdin](https://i18n.boticord.top/).

## Я розробник бібліотеки для вашого API, як мені додати її до вашої документації?

Щоб це зробити, створіть форк репозиторію, додайте потрібну документацію, використовуючи [форматування Markdown](https://docusaurus.io/docs/markdown-features) і надішліть pull request у наш репозиторій. 
Шаблон сторінки в документації для бібліотеки:

```md
---
title: Ім'я
description: Опис
slug: /sdk/ім'я
---

<опис бібліотеки>
```

Форма pull request'а:

```md
Додано інформацію про бібліотеку <назва бібліотеки>

Мова: <мова програмування>
Бібліотека: <назва бібліотеки>

<опис бібліотеки>

Посилання на репозиторій: <посилання на репозиторій>

Посилання на документацію: <посилання на документацію>

Посилання на автора: <посилання на автора>
```

## Переклад

Для перекладу ми використовуємо [Crowdin](https://crowdin.com/). Якщо ви хочете допомогти з перекладом, то перейдіть за [посиланням](https://crowdin.com/project/boticord/).
Зміни файлів у папкі `locale` не допускаються.
