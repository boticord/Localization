# Уступ

Кожны жадаючы можа ўнесці свой унёсак у развіццё праекта. Для гэтага неабходна трымацца наступных правіл:

## Асноўнае

1.  Усе змены павінны быць выразна апісаны ў каміце і pull request'е.
2.  Паводзьце сябе прыстойна. Не выкарыстоўвайце нецэнзурную лексіку і не абражайце іншых удзельнікаў праекта.
3.  Пажадана карыстацца англійскай мовай, паколькі яна з'яўляецца міжнароднай

## Галінкі

-   `main` - галоўная галінка праекта
-   `dev` - галінка распрацоўкі

## Каміты

Выразных правіл для камітаў няма, але ёсць некалькі рэкамендацый:

-   Каміты павінны быць пажадана на англійскай мове
-   Выкарыстоўваць [conventional commits](https://conventionalcommits.org)

### Прыклады

1.  <strike>Changed text</strike> ❌ Каміт не апісвае, чаму быў зменены тэкст, дзе.
2.  **chore(Faq): Added additional answer to the question ** ✅ Ідэальна!

## Дакументацыя

Дакументацыя павінна быць напісана на рускай мове. Для напісання дакументацыі выкарыстоўваецца [Docusaurus](https://docusaurus.io/).
Калі вы жадаеце змяніць дакументацыю на іншай мове, то выкарыстоўвайце [Crowdin](https://i18n.boticord.top/).

## Я распрацоўшчык бібліятэкі для вашага API, як мне дадаць яе ў вашу дакументацыю?

Каб гэта зрабіць, стварыце форк рэпазіторыя, дадайце неабходную дакументацыю, выкарыстоўваючы [фармаціраванне Markdown](https://docusaurus.io/docs/markdown-features) і адпраўце pull request у наш рэпазіторый. 
Шаблон старонкі ў дакументацыі для бібліятэкі:

```md
---
title: Імя
description: Апісанне
slug: /sdk/імя
---

<апісанне бібліятэкі>
```

Форма pull request'а:

```md
Дадана інфармацыя аб бібліятэцы <назва бібліятэкі>

Мова: <мова праграмавання>
Бібліятэка: <назва бібліятэкі>

<апісанне бібліятэкі>

Спасылка на рэпазіторый: <спасылка на рэпазіторый>

Спасылка на дакументацыю: <спасылка на дакументацыю>

Спасылка на аўтара: <спасылка на аўтара>
```

## Пераклад

Для перакладу мы выкарыстоўваем [Crowdin](https://crowdin.com/). Калі вы жадаеце дапамагчы з перакладам, то праходзьце па [спасылцы](https://crowdin.com/project/boticord/).
Змена файлаў у папцы `locale` не дапускаецца.
